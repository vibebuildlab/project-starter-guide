# Markdown Lint Waivers

This document explains acceptable markdownlint violations in this repository.

## Line Length (MD013)

**Files:** README.md, BACKLOG.md, CHANGELOG.md
**Reason:** Documentation files with long URLs, complex descriptions, and badge shields

## Duplicate Headings (MD024)

**Files:** CHANGELOG.md, BACKLOG.md
**Reason:** Standard practice in changelogs to have repeated section headings like "### Added", "### Fixed" across different versions

## Status

These waivers are acceptable for production-perfect standard as they don't affect code quality, security, or functionality.
